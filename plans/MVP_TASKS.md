# git-review-tool MVP タスク分割

## MVP スコープ

MVPでは以下の機能を実装します：
1. 単一コミットの差分取得と表示
2. Hunk単位のコメント保存
3. Hunk単位のレビュー完了状態管理
4. Hunk識別スキーム（同一性判定の基本実装）

### 1.1 プロジェクト構成とツール選定
- [ ] 言語・フレームワークの決定（Node.js/Python/その他）
- [ ] 本体実装用ディレクトリ構造作成（src/、tests/、docs/）

- [ ] CLIアプリケーション構造定義

## フェーズ 2: Git 統合 (フェーズ2)

### 2.1 Diff 取得機構
- [ ] git コマンド実行ラッパー実装
- [ ] `git show` / `git diff` で対象コミットのdiffを取得
  - 単一コミット指定モード
  - 2コミット間差分モード
- [ ] エラーケース処理（無効なコミット、空の差分など）
- [ ] Unified Diff形式での出力確認

### 2.2 Hunk 解析・識別
- [ ] Diff パーサー実装（Unified Diff形式を解析）
- [ ] Unified Diff から以下を抽出：
  - ファイルパス
  - Hunk ヘッダー情報（変更行番号など）
  - Hunk のコンテンツ（追加行、削除行）
- [ ] テスト用フィクスチャ（既知のgitリポジトリ）作成

### 2.3 Hunk Hash 生成スキーム
- [ ] Hunk 内容からの決定論的ハッシュ生成アルゴリズム実装
  - ハッシュ対象：ファイルパス + Hunk コンテンツ（行番号は不含）
  - ハッシュ関数：SHA256 など決定的なもの
- [ ] 同一コミットでの同一 Hunk は同一 Hash を得ることを確認
- [ ] ユニットテスト作成

**成果物**: git diff 取得、Hunk 抽出、Hash 生成が動作する実装

## フェーズ 3: ストレージ層 (フェーズ3)

### 3.1 データモデル定義
- [ ] レビューセッションスキーマ定義

- [ ] Hunk コメントスキーマ定義
  - hunk_hash, hunk_id, comment_text, line_number (行単位コメント用), created_at, updated_at
- [ ] Hunk レビュー状態スキーマ定義
  - hunk_hash, is_reviewed, session_id, reviewed_at

### 3.2 永続化メカニズム選定・実装
- [ ] SQLite vs JSON ファイル の選定
- [ ] データベース初期化処理
- [ ] マイグレーション管理（将来の拡張に対応）

### 3.3 ストレージ API 実装
- [ ] `saveHunkComment(hunkHash, comment)` - Hunk コメント保存
- [ ] `getHunkComment(hunkHash)` - Hunk コメント取得

- [ ] `getHunkReviewStatus(hunkHash)` - レビュー状態取得
- [ ] `deleteHunkComment(hunkHash)` - コメント削除
