# git-review-tool MVP タスク分割

## MVP スコープ

MVPでは以下の機能を実装します：
1. 単一コミットの差分取得と表示
2. Hunk単位のコメント保存
3. Hunk単位のレビュー完了状態管理
4. Hunk識別スキーム（同一性判定の基本実装）

## フェーズ 2: Git 統合 (フェーズ2)

### 2.1 Diff 取得機構
- [x] git コマンド実行ラッパー実装
- [ ] `git show` / `git diff` で対象コミットのdiffを取得
  - 単一コミット指定モード
  - 2コミット間差分モード
- [x] エラーケース処理（無効なコミット、空の差分など）
- [x] Unified Diff形式での出力確認

### 2.2 Hunk 解析・識別
- [x] Diff パーサー実装（Unified Diff形式を解析）
- [x] Unified Diff から以下を抽出：
  - ファイルパス
  - Hunk ヘッダー情報（変更行番号など）
  - Hunk のコンテンツ（追加行、削除行）
- [ ] テスト用フィクスチャ（既知のgitリポジトリ）作成

### 2.3 Hunk Hash 生成スキーム
- [x] Hunk 内容からの決定論的ハッシュ生成アルゴリズム実装
  - ハッシュ対象：ファイルパス + Hunk コンテンツ（行番号は不含）
  - ハッシュ関数：SHA256 など決定的なもの
- [x] 同一コミットでの同一 Hunk は同一 Hash を得ることを確認
- [x] ユニットテスト作成

**成果物**: git diff 取得、Hunk 抽出、Hash 生成が動作する実装
